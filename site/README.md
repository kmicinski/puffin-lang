# The Puffin website

This directory builds the GitHub Pages site at
<https://kmicinski.github.io/puffin-lang/> from sources that already
live in the repo — nothing here duplicates content:

- `index.html` is rendered from the top-level `README.md` (with a hero
  header added);
- `docs/*.html` are rendered from `docs/*.md`, one page per reference
  doc, all through the same shell and s-expression highlighter;
- `docs/tutorial.html` and `docs/stdlib.html` (already HTML — the
  stdlib page is generated by `tools/gen-stdlib-html.rkt`) are copied
  verbatim, so their sibling links keep working;
- `examples/index.html` is rendered from `examples/README.md`.

Links between rendered pages are rewritten `.md → .html`; links into
the source tree (directories, `.puf` files, scripts) point at GitHub.

Build locally:

```
cd site && npm ci && npm run build   # writes _site/ at the repo root
python3 -m http.server -d ../_site   # preview
```

Deployment is `.github/workflows/pages.yml`: every push to `main`
rebuilds and deploys. One-time repo setup: **Settings → Pages → Source
= "GitHub Actions"**.

The playground (`web/`) is *not* part of the Pages build yet — its VM
artifacts (`tools/gen-web-vm.sh`) need the full bootstrap toolchain,
which CI doesn't have. Adding a hosted playground under `/play/` is a
natural follow-on once the wasm artifacts are buildable (or committed)
in CI.
